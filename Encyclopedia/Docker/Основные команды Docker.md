# Основные команды Docker


### Проверка версии и работоспособоности самого 
```
docker -version
```

### Создать образ из Dockerfile
```
docker build -t имя_образа .
```
Ключ `-t` означает, что мы задаём тэг (имя) этому образу
`.` означает, что мы ищем `Dockerfile` в текущей директории терминала


## Образы

- Просмотреть все образы
```
docker images
```


- Удалить образ (если по нему нет контейнеров)
```
docker rmi имя_образа
```

- Скачать образ с docker hub
```
docker pull образ:тег
```


## Контейнеры
- Просмотреть все запущенные контейнеры
```
docker ps
```

- Просмотреть вообще все контейнеры
```
docker ps -a
```

- Запустить **новый** контейнер
```
docker run —-name имя_контейнера образ
```
При запуске Вы сразу попадаете в терминал контейнера
Указанный образ скачивается автоматически, если ещё не скачан

- Если не хотите сразу попадать в терминал контейнера, добавьте аргумент
```
-d
```

- Запустить уже существующий контейнер в фоне
```
docker start имя_контейнера
```

- Остановить контейнер
```
docker stop имя_контейнера
```

- Принудительно остановить контейнер
```
docker kill имя_контейнера
```

- Удалить остановленный контейнер 
```
docker rm имя_контейнера
```

## Логи и отладка
- Просмотр логов. Логи приложения - это то, что приложение выводит в терминал (принтит)
```
docker logs имя_контейнера
```

- Просмотр логов в реальном времени 
```
docker logs -f имя_контейнера
```

- Зайти в терминал контейнера
``` 
docker exec -it имя_контейнера bash
```

- Выйти из терминала контейнера
```
exit
```

- Удалить неиспользуемые данные (образы, контейнеры, кэш и пр.) **АККУРАТНО**
```
docker system prune
```
