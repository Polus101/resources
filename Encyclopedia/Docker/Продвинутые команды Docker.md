# Основные команды Docker

## Аргументы запуска нового контейнера

Аргументы указываются перед указанием образа. 
Всё, что идёт после - переопределение команды запуска (CMD из Dockerfile)
```
docker run аргументы образ командазапуска
```

Простой запуск **нового** контейнера
```
docker run —name имя_контейнера образ
```

<hr></hr>

- Если контейнер должен быть доступен по сети извне, добавьте аргумент `-p`
*(По умолчанию из контейнера можно выйти в сеть. Но вот из сети достучаться до контейнера нельзя. Чтобы контейнер был открыт на вход извне, нужно "пробросить" порт контейнера к реальному порту вашего компьютера)*
```
-p порт_устройства:порт_контейнера
```

- Если контейнер должен использовать внешнюю папку
Папка внутри контейнера и *внешняя* папка на компьютере будут связаны. При удалении такого контейнера данные всё равно останутся во внешней папке на компьютере. Удобно для контейнеров баз данных, например. Контейнер БД удалили, а данные БД остались на компьютере
Или удобно, когда несколько контейнеров должны использовать одну общую внешнюю папку и работать с общими файлами (json`ками, например). Хотя, такого, конечно, лучше избегать

```
-v путь_внешней_папки:путь_папки_в_контейнере
```


- Если контейнер должен использовать **том**
Тоже самое, но использоваться будет том - это та же папка, но хранящаяся во внутренней структуре докера. 
Этот подход чуть удобнее, потому что не надо прописывать путь. И немного надёжнее, так как простую папку на компьютере можно и удалить случайно)))
```
-v название_тома:путь_папки_в_контейнере
```

- Если контейнер должен работать в пользовательской сети
```
--network my-site-network
```

## Пользовательские сети
Если контейнеры должны общаться между собой, придётся объединять их единой "виртуальной локальной сетью" (или открывать контейнер во внешний контур аргументов `-p`, что не всегда является удачной идеей)

Сеть предварительно необходимо создать:
```
docker network create my-site-network
```

Просмотреть все сети:
```
docker network ls
```

Удалить сеть:
```
docker network rm my-site-network
```

Подключить к сети созданный ранее контейнер
*(сработает, даже если контейнер запущен)*
```
docker network connect название-сети название-контейнера
```

Отключить от сети созданный ранее контейнер
*(сработает, даже если контейнер запущен)*
```
docker network disconnect название-сети название-контейнера
```
