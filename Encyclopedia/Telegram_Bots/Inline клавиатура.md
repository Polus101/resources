## Особенности использования
- В отличие от Reply клавиатур, при нажатии на кнопку, не отправляется текст в чат. Текст у кнопок - это просто текст. Он не выполняет больше никакой роли
- Inline клавиатуры прикрепляются непосредственно к сообщению бота

## Как создать 
Правила задания кнопок всё те же. Один общий класс клавиуатуры, в который указывается список списков кнопок. Каждый внутренний список - отдельная строка кнопок

Меняются только названия классов, аргумент `keyboard` сменился на `inline_keyboard` и у каждой кнопки добавился какой то аргумент `callback_data`

```python
my_inline_keyboard = InlineKeyboardMarkup(inline_keyboard=[
    [
        InlineKeyboardButton(text="📊 Кнопка 1", callback_data="btn1"),
        InlineKeyboardButton(text="✏️ Кнопка 2", callback_data="btn2")
    ],
    [
        InlineKeyboardButton(text="👁 Кнопка 3", callback_data="btn3")
    ]
])
```

**callback_data** - то, какую информацию несёт кнопка. Именно по ней мы будем отслеживать нажатия на кнопки. 

По простому `callback_data` - это названия кнопок. Есть текст кнопки, который на ней отображается. А есть название кнопки - это и есть `callback_data`

## Стилизация
Работает так же, как и в Reply кнопках. Смайлики и 3 доступных цвета:
- primary - Синий
- success - Зелёный
- danger - Красный

Для указания стиля добавьте аргумент `style` в аргументы кнопки
```python
InlineKeyboardButton(text="📊 Кнопка 1", callback_data="btn1", style="success")
```


Так же можно указать кастомные эмодзи (для обладателей Premium). Добавляется аргументом `icon_custom_emoji_id`. Его можно указывать в связке со `style` или без него
```python
InlineKeyboardButton(
    text="📊 Кнопка 1", 
    callback_data="btn1", 
    style="success", 
    icon_custom_emoji_id="EMODJI_ID")
```

## Привязка к сообщениям
Привязка абсолютно такая же - просто указывается аргумет `reply_markup` у функции отправки сообщения. В аргумент идёт клавиатура. Она "приклеится" к этому сообщению

```python
    await message.answer("Добро пожаловать в бота", reply_markup=my_inline_keyboard)
```

## Обработка нажатий
Обработка нажатий тоже очень похожая. Чуть чуть меняется декоратор и аргумент функции с `message` на `callback`
```python
from aiogram.types import 

@router.callback_query(F.data == "btn1")
async def handle_btn1(callback: CallbackQuery):
    await callback.answer("Нажата кнопка 1")
    await callback.message.answer("Обратное сообщение")
```
**ВАЖНО** `await callback.answer()` должен быть в **каждом** хэндлере. Иначе кнопка в боте будет выглядеть "зависшей" после её нажатия

## Работа с объектом callback
`callback` здесь - замена `message` и выполняет примерно ту же роль. Через него можно узнать, кто нажал кнопку, что это за пользователь или можно отправлять сообщения в ответ

#### Получение пользователя, который нажал кнопку. 
Внутри него есть всё о пользователе. Как и было раньше
```
callback.from_user
```

#### Получить callback data
Это та самая callback_data, которая указывается при создании кнопок
```
callback.data
```

#### Ответ на сообщение
Это **не ответное сообщение** - это всплывающее окошко с тектом

**ВАЖНО** `await callback.answer()` должен быть в **каждом** хэндлере. Иначе кнопка в боте будет выглядеть "зависшей" после её нажатия
```
await callback.answer("Ответ на нажатие кнопки")
```

#### Ответное сообщение
Это буквально тот же метод, что и был раньше - `message.answer()`
```
await callback.message.answer("Обратное сообщение")
```

#### callback.messsage
это будет тот же объект `message`, с которым мы работали ранее. Но учтите, что в нём лежит сообщение, к которому привязаны inline кнопки. А привязаны они к сообщению бота.

А значит, код ниже будет выдавать id бота, а не пользователя:


```
callback.message.from_user.id
```
