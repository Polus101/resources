# Reply Keyboard
В телеграм ботах есть два вида клавиатур. Один из них - `Reply` клавиатуры. Они проще в освоении и использовании, но выглядят все так же хорошо и выполняют все те же функции.
`Reply` клавиатура - это набор кнопок, который "висит" внизу интерфеса - сразу под полем для ввода сообщения

## Особенности использования
1) Отличие `Reply` клавиатур в следующем: когда пользователь нажимает на кнопку, от его имени в чат с ботом отправляется текстовое сообщение с текстом кнопки. Получается, нажатия на такие кнопки можно обрабатывать как обычные текстовые сообщение. Через фильтр `F.text == "Кнопка 1"`
2) `Reply` клавиатуры "прикрепляются" к какому то сообщению бота. Хотя этого и не видно пользователю (ведь клавиатура отображается внизу), она на самом деле закреплена за сообщением. А значит, если удалить сообщение от бота, к которому была привязана клавиатура, то и клавиатура внизу экрана пропадет
Это накладывает определенные ограничения при работе с ними (второй тип клавиатур эту проблему не решает - в нем они тоже "привязываются" к сообщениям бота)
3) Если вы хотите сделать вложенное меню (а вы хотите), вам будет необходимо описывать несколько клавиатур. По переменной на каждое вложенное меню
4) Этот вид клавиатуры засоряет чат (так как дублирует в чат все нажатые пользователем кнопки). Чтобы этого хоть как то избежать, вы можете удалять сообщение пользователя в момент обработки нажатия им кнопки

## Как создать 
1) Переменные, в которых вы будете описывать клавиатуры, будут громосткими. Так что создайте в проекте отдельный файл для всех клавиатур, которые будут у вас в боте. Назовите его, например, `keyboards.py`
2) В нем из `aiogram.types` импортируйте два класса: `ReplyKeyboardMarkup` и `KeyboardButton`
```python
from aiogram.types import ReplyKeyboardMarkup, KeyboardButton
```
3) Создайте в этом файле переменную, в которой будете хранить клавиатуру. 
```python
main_menu_keyboard = ReplyKeyboardMarkup(keyboard=[
    [KeyboardButton(text='📊 Кнопка 1'), KeyboardButton(text='✏️ Кнопка 2')],
    [KeyboardButton(text='👁 Кнопка 3')],
],
resize_keyboard=True)
```

Здесь мы видим, что клавиатура - это объект класса `ReplyKeyboardMarkup`, в котором указано два аргумента:
- `keyboard` - аргумент, в котором список наших кнопок (подробнее о нем ниже)
- `resize_keyboard` - задавая ему значение `True`, мы заставляем телеграм адаптировать размер кнопок под экран устройства пользователя. Если его не указать или задать `False` (и так стоит по умолчанию), то клавиатура будет фиксированного (огромного) размера 

## Стилизация
Помимо того, что в кнопках можно использовать смайлики, можно так же задавать цвет кнопкам.

Но пока есть только 3 доступных цвета
- primary - Синий
- success - Зелёный
- danger - Красный

Для указания стиля просто добавьте аргумент `style` в аргументы кнопки
```python
KeyboardButton(text='📊 Кнопка 1', style="success")
```

Так же можно указать кастомные эмодзи (для обладателей Premium). Добавляется аргументом `icon_custom_emoji_id`. Его можно указывать в связке со `style` или без него
```python
KeyboardButton(text='📊 Кнопка 1', style="success", icon_custom_emoji_id="EMODJI_ID")
```

#### Правила задания кнопок (аргумент keyboard)
В аргумент `keyboard` передается список кнопок. Вот по каким правилам:
- Список должен хранить в себе другие списки
- Каждый такой вложенный список - очередная строка с кнопками
- Во вложеных списках должны храниться объекты класса `KeyboardButton` с аргументом `text`
- Аргумент `text` - это текст, который отображается на кнопке. Он же будет отправлен чат, когда пользователь нажмет на кнопку
Так в примере выше видно, что в **первом** вложенном списке две кнопки (`Кнопка 1` и `Кнопка 2`). И они отображаются на **первой** строке клавиатуры.
А во **втором** вложенном списке одна кнопка (`Кнопка 3`) и она отображается на **второй** строке клавитуры


## Привязка к сообщениям
После создания переменной с кнопками ее необходимо привязать к какому то сообщению. 
Проще говоря, указать, в момент какого сообщения бота эта клавиатура будет появляться перед пользователем

И делается это проще, чем кажется. Всего лишь нужно указать аргумент `reply_markup`в функции `answer` и дать на вход свою переменную с клавиатурой 

```python
from aiogram.types import Message

@dp.message(F.text=="/start")
async def cmd_start(message: Message):
    await message.answer("Добро пожаловать в бота", reply_markup=main_menu_keyboard)
```

В итоге, если в бота зайдет пользователь и введет команду `/start`, бот пришет ему сообщение `Добро пожаловать в бота` и внизу у пользователя появится наша клавиатура

Соответственно, если в какой то момент вам нужно отобразить пользователю другую клавиатуру (например, если у вас вложенное меню), вы так же на какое то сообщение привяжете другую клавиатуру

**Учтите: если пользователь удалит сообщение, к которому привязана клавиатура, она у него пропадет!**

## Обработка нажатий
Как и было сказано, обрабатываются нажатия на такие кнопки как обычный текст. Так как нажатие на кнопку отправляет в чат от имени пользователя текст кнопки
В примере выше было три кнопки. Вот функция обработчик первой кнопки:
```python
from aiogram.types import Message

@dp.message(F.text=='📊 Кнопка 1')
async def button_one(message: Message):
    await message.delete() # удаляет сообщение пользователя в котором содержится просто текст кнопки
    await message.answer("Зафиксировано нажатие на кнопку 1")
```
