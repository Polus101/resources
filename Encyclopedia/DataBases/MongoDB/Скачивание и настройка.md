# Скачивание и настройка MongoDB

<hr>

## Скачивание Для MacOS и Linux
### Скачивание MongoDB сервера
1) Установите `brew` - это терминальная утилита для скачивания и управления пакетов. Зайдите на официальный [сайт](https://brew.sh) и скопируйте команду для вашей системы
2) Перейдите на [сайт](https://www.mongodb.com/try/download/community). Вы попадёте в раздел **MongoDB Community Server Download**. 
3) Выберите свою версию ОС и скачайте сервер, скопировав показанную команду в терминал

- Запуск осуществляется командой в терминале `brew services start mongodb-community@8.0`
- Остановка осуществляется командой в терминале `brew services stop mongodb-community@8.0`

### Скачивание графической оболочки
Для "удобного" управления базой данных можно скачать `MongoDB Compass` - это графическая система управления базой данных (`СУБД`). Она необязательна и, когда вы освоитесь, заходить в неё вы будете крайне редко. Но для начала она будет удобна

1) Перейдите на [сайт](https://www.mongodb.com/try/download/compass). Вы попадёте в раздел **MongoDB Compass Download (GUI)**. 
2) Выберите последнюю `stable` версию программу и свою версию ОС и скачайте СУБД.

## Скачивание Для Windows
### Скачивание графической оболочки вместе с сервером
Если вы разрабатываете на Windows, то вам повезло меньше - настройка будет сложнее, а проблем больше. Mongo - серверная база данных и хорошо работает на серверных ОС (Linux и, как следствие, MacOS)

`MongoDB Compass` - это графическая система управления базой данных (`СУБД`).
**Для Windows она обязательна**. На сайте Mongo в принципе нет опции скачивания сервера базы данных без неё
`Compass` скачивается с уже вшитым сервером, от чего его отдельная настройка превращается в танцы с бубном.

Сложная разработка подразумевает наличие выделенного сервера с БД под управлением Linux. "Крутить" MongoDB локально на своей Windows машине - плохая идея
Однако, сильно не пугайтесь - вскоре вы дойдёте и до этого уровня.
**А пока разработка простая, можно устанавливать прямо на свой ПК с Windows**

1) Перейдите на [сайт](https://www.mongodb.com/try/download/compass). Вы попадёте в раздел **MongoDB Compass Download (GUI)**. 
2) Выберите последнюю `stable` версию программы и свою версию ОС и скачайте СУБД.

## Настройка MongoDB Compass
**Этот шаг обязателен, если вы находитесь под Windows**

Открываем MongoDB Compass - ярлык находится на рабочем столе.

Нас встречает окно программы, в котором первым делом необходимо установить подключение к MongoDB серверу.
Он у вас пока будет локальный, но тем же алгоритмом можно подключиться к Mongo, если она находится где то на другом компьютере или виртуальном сервере
<p align="center">
<img src="https://github.com/Polus101/resources/blob/master/Encyclopedia/DataBases/MongoDB/img/new_connection.png" style="width:60%"/>
</p>

<hr>

- В появившемся окне настройки подключения необходимо ввести адрес Mongo сервера. Так как он у вас локальный, мы прописываем `localhost` и стандартный порт, на котором работает Mongo сервер `27017`
```
mongodb://localhost:27017/
```

- Так же можно задать какое нибудь название для этого подключения. Оно будет отображаться толко у вас на компьютере
- И можно задать цвет, которым будет отмечено это подключение

<p align="center">
<img src="https://github.com/Polus101/resources/blob/master/Encyclopedia/DataBases/MongoDB/img/connection_params.png" style="width:100%"/>
</p>

**В продвинутые настройки пока не идём. Их можно будет настроить после, когда освоитесь с БД**


<hr>

После проделанных процедур слева мы увидим наш Mongo сервер. А вернее, все базы данных, работающие на нём

Здесь уже есть 3 стандартных: `admin`, `config` и `local`
**Удалять их нельзя** - это базы данных, которые использует сам сервер для своих нужд

<p align="center">
<img src="https://github.com/Polus101/resources/blob/master/Encyclopedia/DataBases/MongoDB/img/databases.png" style="width:60%"/>
</p>

<hr>

Нам же необходимо создать свою базу данных под проект. Под каждый проект будет своя база данных

*P.S. Для больших проектов можно создавать несколько баз данных для удобства и бОльшей отказоусточивости. А так же развёртывать несколько серверов Mongo*

Нажимаем **+** для создания новой базы данных

<p align="center">
<img src="https://github.com/Polus101/resources/blob/master/Encyclopedia/DataBases/MongoDB/img/plus_db.png" style="width:60%"/>
</p>


- В появившемся окне задаём имя для базы данных (оно может быть созвучно с названием вашего проекта)
- Так же адаём название для первой `коллекции`
- И создаём базу данных

**Коллекция** - это набор данных. Например, в телеграм боте или на сайте 100% будут данные о пользователях. Значит, будет резонно создать коллекцию `users` для хранения в ней информации о пользователях

<p align="center">
<img src="https://github.com/Polus101/resources/blob/master/Encyclopedia/DataBases/MongoDB/img/db_settings.png" style="width:60%"/>
</p>

P.S. Чуть чуть про названия: в Mongo нет чётких правил как называть базы данных и коллекции. Главное, чтобы не было пробелов. Так что можете назвать базу данных `my_telegram_bot`, как переменную в python. А можете, например `myTelegramBot`, как переменные в java - как вам больше нравится
Даже в системной базе данных `config` самой Mongo используются разные стили названий
